# Skill: Sprint Slicing

## Purpose

Group backlog items into sprint-sized increments that end in a runnable and reviewable outcome.

## Use this skill when

- backlog items already exist
- work must be planned into increments
- each sprint should end with a usable partial implementation

## Inputs

- task list
- dependencies
- acceptance criteria
- technical constraints

## Output contract

Produce:

- sprint candidates
- sprint goals
- runnable increment definition per sprint
- major risks per sprint

## Rules

- every sprint must end with something testable
- avoid stacking invisible infrastructure work without validation
- prefer vertical slices over isolated technical layers where possible
- identify what can actually be demonstrated at sprint end

## Escalate when

- no runnable increment can be defined
- dependency order blocks incremental delivery
- acceptance path is unclear
