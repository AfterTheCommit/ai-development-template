# Skills

## What a skill is

A skill is a narrow, structured capability intended for repeatable and more deterministic subtasks.

## Current core skills

- prompt-to-spec
- task-decomposition
- sprint-slicing
- dod-generator

## Why skills exist

Skills are used when a task:

- has clear inputs and outputs
- benefits from standard structure
- should be repeatable across projects

## Implementation note

At this stage, skills are documented as repository assets and templates.
Further runtime integration will be added later.
