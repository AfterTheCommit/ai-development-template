# Label Strategy

## Purpose

Labels provide lightweight classification beyond issue templates and project fields.

## Recommended labels

### Type
- type:feature
- type:bug
- type:spike
- type:task

### Area
- area:backend
- area:frontend
- area:database
- area:docs
- area:infra
- area:mixed

### State / workflow
- state:blocked
- state:needs-review
- state:needs-clarification

### AI workflow
- ai:claude
- ai:human-review-required

### Priority
- priority:p0
- priority:p1
- priority:p2
- priority:p3

## Rules

- use labels for quick filtering
- use GitHub Project fields for structured reporting
- do not overload labels with information better stored in fields
